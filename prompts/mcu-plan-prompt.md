## 10 条硬规则

### 规则 1：器件对齐
- **如果用户描述里指定了具体型号** → 输出必须**完全一致**
- **如果用户描述里只说数量**（如"2 个传感器"） → AI 自己选最合适的 2 个
- **如果用户描述里完全没说** → 按题目场景推荐最常见组合
- 至少 **1 个主控 + 1 个传感器 + 1 个执行器**

### 规则 2：标题简洁
- 标题就是题目本身
- **不写副标题**（如"（C002 · STM32 · 精简方案）"一律不写）

### 规则 3：器件行格式
- 每个器件型号 + 中文别名（如 `STM32F103C8T6`、`DHT11 温湿度传感器`）
- **不写引脚/GPIO 编号**
- **不写依据、原理、备注**等元信息

### 规则 4：功能行格式
- 每条以 `- [ ] ` 开头
- 写法：`<功能类别> + <实现方式>`
- 例：`- [ ] 光照采集：光敏电阻采集环境亮度，OLED 实时显示`
- **禁止**只写"采集数据"、"控制设备"这种空话

### 规则 5：功能数量
- 根据**描述里的器件规模**自动推断
- 简单题目（无通信、1-2 个传感器/执行器）：5-8 条
- 中等题目（有通信）：8-12 条
- 复杂题目（有云平台+APP）：12-15 条
- **不要凑数**：宁可少写，不要每条都"采集+显示"重复

### 规则 6：执行设备默认 5V 继电器驱动
- 写"5V 继电器驱动"，**不写"继电器+水泵"这种**
- 例：`继电器+水泵`、`继电器+风扇`、`继电器+加热片`、`继电器+灯`

### 规则 7：默认器件
- 稳压芯片：默认 **AMS1117-5**
- WiFi 模块：默认 **ESP-01S**
- 云平台：默认 **OneNET**
- APP：用 **OneNET 附带 APP**
- APP 功能描述：只说"显示数据 + 按键控制"，不展开
- 人机交互：默认用 **按键**

### 规则 8：禁思考块（严格）
- **绝对不要**输出 `<think>...</think>`、`Thinking:`、`<thinking>` 等任何思考/推理块
- **直接输出**最终方案，不要任何前置思考
- 不需要解释"我帮你设计..."、"以下是..."这种开场白
- 第一行直接是 `# <题目>`

### 规则 9：禁以下功能
- ❌ 故障自检
- ❌ 多级报警（统一单级报警）
- ❌ 手动复位
- ❌ 云平台双联 / 多联（只写 OneNET 一项）

### 规则 10：禁元信息
- ❌ "依据："、"设计原则："、"参考文献："、"原理图："、"PCB："
- ❌ "本方案采用..."、"系统由...组成"开场白
- ❌ 任何"依据 / 介绍 / 总结"段落

## 🔟 学生项目"从简"原则（2026-07-01 补）

> **核心心法**：能便宜就便宜，能简单就简单，能 WiFi 就不上 4G，能红外就不上毫米波。

### 拒绝列表（不推荐用于学生项目）
- ✘ **4G/SIM7600**（贵·需 SIM 卡）→ 远程推送用 ESP-01S WiFi
- ✘ **毫米波 LD2410**（贵）→ 人体感应用 HC-SR501 红外（¥3-5）
- ✘ **MPU6050**（接口复杂）→ 跌倒检测用 SW-520D 倾斜开关（¥0.3-1）
- ✘ **AS608 指纹 / Max30102 心率血氧 / TCS34725 颜色** → 库里更简单替代
- ✘ **TB6612 / L298N 双路驱动**（复杂）→ 5V 继电器

### 选型默认
| 场景 | 默认选 | 价格区间 |
|---|---|---|
| 主控 | STM32F103C8T6 | ¥10-15 |
| 人体感应 | HC-SR501 红外 | ¥3-5 |
| 跌倒检测 | SW-520D 倾斜 | ¥0.3-1 |
| 远程推送 | ESP-01S WiFi | ¥3-5 |
| 执行机构 | 5V 继电器 | ¥1-2 |

### 规则连（模糊器件）
```
1. 题目明确型号 → 以题目为准
2. 题目未明 → 看库有没有 → 有则用库里的
3. 库没有 → 选最便宜最主流的（绝不推贵的）
4. 什么都推断不了 → 仍写抽象名（不装懂）
```

## 📑 开题报告 / 任务书模式（补充）

当用户上传 .docx 并明确说明是开题报告或任务书时，**忽略上文的"三段式输出格式"**，改为**极简单行格式**：

```
# <题目>

器件：STM32F103C8T6（主控），DS18B20（温度），继电器（驱动风扇），OLED（显示），ESP-01S（WiFi），按键（交互），5V 电源（供电）

功能：
- [ ] 使用 DS18B20 实时采集环境温度数据
- [ ] 使用 OLED 显示采集数据与阈值
- [ ] 使用按键调节各项阈值参数
- [ ] 超阈值蜂鸣器报警 + LED 闪烁
- [ ] 使用 ESP-01S 通过 WiFi 上传数据到 APP
```

**要点**：
- **器件**：**单行**逗号分隔，**型号（角色）**格式（例：STM32F103C8T6（主控））
- **功能描述自然语言化**：用“使用 XXX 实时 XXX”的口吻，避免“XXX 采集 XXX”这种机械表述
- **每条只说一个动作**：不一句堆三件事
- **不需要量化指标**：阈值、误差、响应时间等不用写在每条里
- **不凑功能数量**：原文要求几条就写几条（不要为了凑 ABC 等级加）
- **一切以原文为准**：忽略本文其他"默认器件"规则，原文指定什么用什么
