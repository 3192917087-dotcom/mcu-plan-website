## 四、逐模块填充（8 模块生成公式）

### 1. 选题背景 / 立题依据
**公式**：场景行业现状 → 传统方式痛点 → 单片机技术适配性 → 引出本设计。

**🚫 背景章节铁律（v0.5 · 2026-07-04 19:24 拍板）**：
- ❌ **不列器件清单**（不出现 STM32F103C8T6 / DHT11 / ESP-01S 等具体型号）
- ❌ **不列功能清单**（不出现"本系统实现 XX 采集 + YY 显示"这类具体功能点）
- ✅ 只用通用描述："嵌入式单片机技术、低成本传感器与 WiFi 通信模组在过去几年快速普及"
- 作用：场景铺垫 + 痛点引出，详细方案留给"技术方案"章节

**真实数据约束**：
- ❌ 不编造具体数字（"3500 万户"/"200 亿元"等）
- ✅ 用通用描述："家庭观赏鱼养殖用户众多，市场规模庞大"
- 末尾标注"具体数据以实际调研为准"

### 2. 研究 / 设计意义
**公式**：实践意义（解决具体问题）+ 技术意义（提供方案参考）。
- 实践意义：紧扣功能说清能给场景带来什么
- 技术意义：紧扣器件方案说明对同类设计的参考价值，**不夸大理论贡献**

### 3. 国内外研究现状
**公式**：国内方案分类梳理 → 国外方案特点简述 → 现有方案共性不足 → 引出本设计改进方向。
- 国内按"主控类型 / 技术路线"分类
- **真实性约束**：
  - ❌ 不编造具体学者姓名
  - ✅ 允许通用指代："国内学者在 [方向] 上提出多种方案"
  - ✅ 句式：「[方向] 的研究主要聚焦于 XX，技术路线分为...」
  - 末尾标注："本节为示意性梳理，正式答辩前需补充 3-5 篇近 3 年真实文献"

### 4. 研究目标与主要内容
**研究目标**：1 个总体目标 + 3 个可落地具体目标（硬件 / 软件 / 测试三维度）。

**主要内容**：单片机开发 4 个固定环节：
1. 方案调研与选型论证
2. 系统硬件设计与制作
3. 系统软件设计与开发
4. 系统联调与性能测试

### 5. 技术方案 / 总体方案
**总体架构**：「输入层 → 主控层 → 输出/执行层 + 通信层 + 电源层」。

**硬件方案**：
- 核心器件详写 5-7 项：主控 > 通信 > 显示 > 关键传感器 > 其他传感器
- 关键传感器定义：直接关联核心基础功能的传感器
- 辅助模块按类分述：电源 / 交互 / 执行 / 通信，每类 1 行 + 总括收尾

**软件方案**：开发环境（按主控选：STM32→Keil MDK5；51→Keil C51；Arduino→Arduino IDE）+ 主程序流程（初始化 → 循环采集 → 处理 → 执行）+ 软件功能模块拆分。

> 框图/流程图：开题阶段文字描述清楚结构逻辑即可，标注"系统框图待后续绘制"，**不生成图片**。

### 6. 研究方法
**公式**：选 3-4 种适配单片机的方法，每种绑定本课题具体内容。
- 文献研究法 / 对比选型法 / 软硬件协同设计法 / 实验测试法

### 7. 关键问题
**复杂度量化分档**：

| 复杂度 | 判定标准 | 关键问题模板 |
|---|---|---|
| 基础简单 | 器件 ≤3，无通信 | ① 传感器数据稳定性 → 软件滤波 + 多次采样<br>② 硬件焊接与调试 → 分模块焊接 + 仪器逐点测试 |
| 中等复杂 | 器件 4-10，或含通信 | 在"基础"基础上 + ③ 多模块硬件兼容性与通信稳定性 → 统一协议 + 电平匹配 + 分模块调试 |
| 复杂 | 器件 ≥11，或含闭环控制算法 | 在"中等"基础上 + ④ 核心算法落地与优化 → 简化算法 + 实测调参 + 边界 case 覆盖 |

### 8. 设计特色与创新点

**7.1 关键问题**（按复杂度档位选 2-4 个）

**7.2 设计特色**（合理性说明，非学术 novelty）
- 成本 / 性能 / 开发难度的权衡说明
- 通用成熟器件选型、本科难度定位、毕设周期匹配

**7.3 创新点**（差异化亮点，2 个维度）
- 功能集成：把 XX、XX、XX 功能集于一体
- 场景适配：针对 XX 场景简化操作 / 优化功耗

### 9. 进度安排 / 预期成果 / 可行性分析
标准化模块，基础框架固定，只替换课题要素：
- 进度：调研 → 选型 → 硬件 → 软件 → 联调 → 论文 → 答辩（12-16 周）
- 预期成果：实物成果 + 论文成果 + 能力收获
- 可行性：技术 + 时间 + 经济 三维度
