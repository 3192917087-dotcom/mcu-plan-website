## 🔧 14 步端到端流程

| # | 步骤 | 工具 |
|---|---|---|
| 1 | 用户导入方案文档 | `mammoth.browser.min.js` |
| 2 | 提取题目 / 器件 / 功能 | 正则 |
| 3 | 用户确认 / 直接生成 | UI 表单 |
| 4 | 用户填 API Key | localStorage |
| 5 | 点"🚀 生成论文" | 按钮 |
| 6 | Ch1 绪论 串行生成 | MiniMax M3 (thinking disabled) |
| 7 | Ch2 总方案 | 同上 |
| 8 | Ch3 硬件 | 同上（锁器件） |
| 9 | Ch4 软件 | 同上 |
| 10 | Ch5 测试 | 同上 |
| 11 | Ch6 总结 | 同上（项目总结） |
| 12 | 摘要 + 致谢 + 参考文献 | 同上 |
| 13 | 嵌入 Ch4 流程图 | ASCII + Mermaid 双轨 |
| 14 | 输出 .docx | JSZip + 首行缩进 XML |

## ✅ 防画蛇添足 · 7 项硬约束

1. 锁器件清单（11 器件）
2. Forbidden 型号清单（8 项）
3. 标题锁死
4. 不编造引脚/寄存器/阈值
5. Ch1 唯一允许 [n] 引用
6. Ch6 = 项目工作总结（不是论文总结）
7. 不第一人称

## 📊 输出格式

| 输出 | 路径 |
|---|---|
| 预览 | `<article.result-body>` 内 |
| Markdown | 复制按钮 → 剪贴板 |
| .docx | 下载按钮 → 浏览器 |

**.docx 包含**：

- 12 个 H1（摘要 / Abstract / 目录 / 6 章 / 致谢 / 参考文献）
- 21+ 个 H2
- 10+ 个 H3
- 18+ 个 `[待插入图：...]` 占位符
- 5 个 ASCII 流程图（等宽字体，复制即可用）
- 5 个 Mermaid 流程图（复制到 mermaid.live 渲染）
- 95.8% 正文段首行缩进 2 字符（XML `<w:ind>` 注入）

## 🔬 与 Python 后端对比

| 维度 | 前端 (thesis/) | Python (backend/) |
|---|---|---|
| **执行环境** | 浏览器 | 命令行 |
| **API 调用** | fetch (用户填 key) | urllib (key in api.js) |
| **生成 6 章** | 串行 + UI 进度 | 串行 + 进度（命令行） |
| **.docx 输出** | JSZip (浏览器) | python-docx (python) |
| **首行缩进** | XML 注入 | post-process |
| **学校模板** | 不支持（M1 路径：自定义） | M3 路径（完整复刻封面） |
| **速度** | 8-15 分钟 | 8-15 分钟 |
| **结果** | 15000 字 + 占位符 + 流程图 | 15000 字 + 占位符 + 流程图 + 学校封面 |

## ⚠️ 已知限制

- 不支持学校论文模板（前端无法处理学校 .docx 中的样式覆盖）
- 流程图是 ASCII + Mermaid，**用户需手动渲染 Mermaid**（或复制到 mermaid.live）
- 没有"自动清理第一版 v2 重试"按钮（如果生成效果不理想，用户点"🚀 生成论文"再次触发）

## 📝 用法（用户角度）

```
1. 打开 mcu-plan-website 主页
2. 点 "📄 论文生成"
3. 粘贴 API Key（来自 api.js 第 11 行）
4. 拖入方案文档（.docx）
5. 校对提取的题目/器件/功能
6. 选字数（10000/15000/20000）
7. 点 "🚀 生成论文"
8. 等 8-15 分钟
9. 看预览 / 下载 .docx
10. 用 Word 打开 .docx，手动插入图片替换占位符
```
